# 新手教學：用 GitHub Desktop 或 Claude Code 參與協作

這份文件帶你一步步完成 [README.md](README.md) 裡的協作流程。
不用背指令，照著做就好。先看你想用哪一種工具：

- **GitHub Desktop** → 圖形介面、用滑鼠點，**最推薦新手**。
- **Claude Code** → 在終端機用自然語言請 AI 幫你操作，適合想順便學 CLI 的人。

> 名詞快速理解：
> - **repo**：放程式碼的專案資料夾。
> - **branch（分支）**：你自己的工作區，改東西不會影響別人。
> - **commit**：把這次的修改「存檔」並寫一句說明。
> - **push**：把本機的 commit 上傳到 GitHub。
> - **Pull Request（PR）**：請大家來看你的修改、同意後合併進 main。

---

## 路線 A：GitHub Desktop（推薦新手）

### 0. 安裝與登入
1. 到 https://desktop.github.com 下載安裝。
2. 開啟後用你的 GitHub 帳號登入（File → Options → Accounts）。

### 1. Clone repo（把專案下載到電腦）
1. 上方選單 **File → Clone repository**。
2. 在清單找到 `git-practice`（或貼上網址 `https://github.com/justinzhong123/git-practice`）。
3. 選一個本機資料夾位置，按 **Clone**。

### 2. 建立自己的 branch
1. 左上角點 **Current branch**（目前應該是 `main`）。
2. 按 **New branch**。
3. 名稱填 `feature/你的名字`（例如 `feature/amy`），按 **Create branch**。
   - 之後請確認左上角顯示的是你的 branch，不是 main。
   

### 3. 編輯 members.md
1. 用任何文字編輯器（VS Code、記事本都行）打開資料夾裡的 `members.md`。
2. 找一個空的「成員 N」區塊，填入你的資料，存檔。

### 4. Commit（存檔 + 寫說明）
1. 回到 GitHub Desktop，左側會看到你改了 `members.md`。
2. 左下角 **Summary** 欄填：`feat: add 你的名字`。
3. 按 **Commit to feature/你的名字**。

### 5. Push 並開 Pull Request
1. 按上方 **Push origin**（第一次會是 Publish branch）。
2. 按出現的 **Preview Pull Request** 或 **Create Pull Request**，會跳到瀏覽器。
3. 確認標題與內容，按綠色 **Create pull request**。
   - 注意 base（合併目標）要是 `main`。

### 6. Review 與 merge
1. 等至少 1 位組員在 PR 上按 **Approve**（這個 repo 規定要 1 人同意才能合併）。
2. 若出現 **conflict（衝突）**，見最下方「衝突怎麼辦」。
3. 通過後按 **Merge pull request** 完成。

## 路線 B：Claude Code（終端機 + AI 助手）

Claude Code 可以用講的，但你還是要懂每一步在做什麼。下面每步都附上「你可以這樣對 Claude 說」。

### 0. 前置
- 安裝 Claude Code，並先用 `gh auth login` 或 GitHub 登入過。

### 1. Clone repo
> 對 Claude 說：「幫我 clone https://github.com/justinzhong123/git-practice 到目前資料夾」
對應指令：
```bash
git clone https://github.com/justinzhong123/git-practice
cd git-practice
```

### 2. 建立 branch
> 「幫我建立並切換到 branch `feature/我的名字`」
```bash
git checkout -b feature/你的名字
```

### 3. 編輯 members.md
> 「打開 members.md，在成員 2 的區塊幫我填入：姓名…、系級…、興趣…、一句話介紹…」
（Claude 會直接幫你改檔案，你檢查一下內容是否正確。）

### 4. Commit
> 「把 members.md 加入並 commit，訊息用 `feat: add 我的名字`」
```bash
git add members.md
git commit -m "feat: add 你的名字"
```

### 5. Push 並開 PR
> 「push 這個 branch，然後幫我開一個 Pull Request 到 main」
```bash
git push -u origin feature/你的名字
gh pr create --base main --fill
```

### 6. 等 review、merge
> 「這個 PR 現在的 review 狀態如何？」「有 conflict 的話幫我解決」
通過後：
```bash
gh pr merge --merge
```

## 衝突怎麼辦（conflict）

當兩個人改到「同一個地方」，merge 時就會衝突。別緊張，這很正常。

- **GitHub Desktop**：會跳出衝突檔案，用 VS Code 打開，會看到像這樣的標記：
  ```
  <<<<<<< HEAD
  別人的版本
  =======
  你的版本
  >>>>>>> feature/你的名字
  ```
  把不要的部分（含 `<<<`, `===`, `>>>` 三行標記）刪掉，留下正確內容，存檔，再 commit 一次即可。

- **Claude Code**：直接說「幫我解決 members.md 的 conflict，兩個人的資料都保留」，AI 會幫你處理並說明。

> 預防小技巧：每個人填**不同**的成員區塊，就幾乎不會衝突 🙂

---

## 重要規則（這個 repo 的設定）

- main branch 受保護：**不能直接 push 到 main**，一定要走 PR。
- PR 需要**至少 1 人 Approve** 才能 merge。
- GitHub 預設**不能自己 approve 自己的 PR**，所以請互相幫忙 review。
